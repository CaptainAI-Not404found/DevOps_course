 COPY — копирует файлы из машины разработчика в образ.
RUN — выполняет команду на этапе сборки образа (например, устанавливает зависимости).
EXPOSE — объявляет, какой порт внутри контейнера будет использовать приложение.
CMD — указывает команду, которую контейнер выполнит при запуске (не во время сборки).

Почему requirements.txt копируют раньше остальных файлов:
Чтобы Docker кэшировал слой с установкой зависимостей. Если requirements.txt не меняется, Docker не будет снова устанавливать библиотеки — сборка идёт быстрее. Если же копировать весь проект сразу, любое изменение в коде сбросит кэш, и зависимости будут ставиться заново.
